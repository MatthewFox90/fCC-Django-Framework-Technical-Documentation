Introduction

Django is an open-source framework for web application development based on Python. It provides a range of tools and features that simplify the process of developing a web application, such as support for URL mapping, the model-view-template pattern, support for databases, and much more.

# Import the Django framework
import django
            
# Print the version of Django
print(django.get_version())
  • Import the Django framework
  • Print the version of Django
Installation

To use Django, you must first install Python on your computer. Once Python is installed, you can install Django using the "pip" package manager.

# Install Django using pip
!pip install django
  • Install Django using pip
Creating a project

Once Django is installed, you can create a new project using the command "django-admin startproject project_name". This command will create the basic structure of the project, which includes the configuration files and the folder for the applications.

# Create a new Django project called "myproject"
!django-admin startproject myproject
  • Create a new Django project using the "django-admin startproject" command
Creating an app

Once the project is created, you can create an app within the project using the command "python manage.py startapp app_name". This command will create the basic structure for the app, which includes the configuration files and the folder for the models (Django models represent database tables).

# Go to the project directory
cd myproject

# Create a new Django app called "myapp"
!python manage.py startapp myapp
  • Go to the project directory
  • Create a new Django app using the "python manage.py startapp" command
Configuring the database

Django supports many types of databases, such as MySQL, PostgreSQL, and SQLite. To configure the database for the app, you need to modify the "settings.py" file of the project and set the database variables.

# In the "settings.py" file of the project, set the database variables
DATABASES = {
    'default': {
        'ENGINE': 'django.db.backends.sqlite3',
        'NAME': os.path.join(BASE_DIR, 'db.sqlite3'),
    }
}
  • In the "settings.py" file of the project, set the database variables
Defining models

Models are used by Django to represent database tables. To define a model, you need to create a class that inherits from "models.Model" and define the fields of the model as class variables.

# In the "models.py" file of the app, define a model called "Person"
from django.db import models

class Person(models.Model):
    name = models.CharField(max_length=100)
    age = models.IntegerField()
  • In the "models.py" file of the app, define a model class that inherits from "models.Model"
  • Define the fields of the model as class variables
Running migrations

Once the model classes are defined, you need to run migrations to create the database tables. To do this, you need to use the command "python manage.py makemigrations" to create the migrations and then "python manage.py migrate" to apply them to the database.

# Create the migrations for the app
!python manage.py makemigrations myapp

# Apply the migrations to the database
!python manage.py migrate
  • Create the migrations for the app using the "python manage.py makemigrations" command
  • Apply the migrations to the database using the "python manage.py migrate" command
Defining views

Views are functions that handle HTTP requests and return responses. To define a view, you need to create a function that accepts an HTTP request and returns a response.

# In the "views.py" file of the app, define a view called "home"
from django.shortcuts import render

def home(request):
    return render(request, 'home.html')
  • In the "views.py" file of the app, define a view function
URL mapping

To associate a view with a specific URL, you need to define a URL pattern in the "urls.py" file of the app. The URL pattern specifies the regular expression that the URL should match and the view function that should be called.

# In the "urls.py" file of the app, define a URL pattern for the "home" view
from django.urls import path
from . import views

urlpatterns = [
    path('', views.home, name='home'),
]
  • In the "urls.py" file of the app, define a URL pattern for the view function
Templates

Django uses templates to define the HTML structure of a webpage. You can create a template and use it to render the HTML of a view function by using the "render()" function.

# In the "templates" folder of the app, create a file called "home.html"
<!DOCTYPE html>
<html>
<head>
    <title>Home</title>
</head>
<body>
    <h1>Welcome to the home page!</h1>
</body>
</html>

# In the "views.py" file of the app, render the "home.html" template in the "home" view
from django.shortcuts import render

def home(request):
    return render(request, 'home.html')
  • In the "templates" folder of the app, create a template file
  • In the view function, use the "render()" function to render the template and return it as the response to an HTTP request
